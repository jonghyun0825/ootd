# Step 04. Mock 데이터 기반 화면 구현

## 목표
- Supabase 연동 없이 mock 데이터(`services/mockData.ts`)로 전체 화면을 완성한다.
- 구현 대상: 로그인 화면, 홈 화면, 필터 화면, 업로드 화면(사진별 태그 입력 카드 포함), 아카이브 화면, 상세 화면, 수정 화면.
- 절대 포함하지 않는 것: AI 이미지 분석 기능, 텍스트 검색창.
- 검색/탐색은 오직 태그 선택 필터(계절/카테고리/아이템종류/색상/자유태그)로만 동작해야 한다.

## 완료 기준
- [ ] 로그인 화면: 이메일 입력 UI + 안내 문구 (실제 인증은 아직 mock)
- [ ] 홈 화면: 최근 사진, 계절 빠른 필터, 업로드 버튼, 빈 상태 문구
- [ ] 필터 화면: 계절/카테고리/아이템종류/색상/자유태그 선택 UI, AND 조건 필터링 동작
- [ ] 업로드 화면: 다중 사진 선택, 사진별 태그 입력 카드, 일괄 계절/태그 적용
- [ ] 아카이브 화면: 전체/계절별/카테고리별 보기, 정렬, 백업 내보내기 버튼(mock), 로그아웃
- [ ] 상세 화면: 사진 정보 전체 표시 + 수정/삭제 버튼
- [ ] 수정 화면: 기존 값이 채워진 폼, 저장/취소
- [ ] 텍스트 검색창 및 AI 분석 관련 UI/코드 없음 확인
